Negative Integer Literals

Posted at — Jun 12, 2023
  1. negative numbers are represented as two’s complement and,
  2. implicit type conversion of a 32-bit integer to 64-bit integer preserves the value.

With the knowledge of above two points, you’d expect that the functions f and g below would return the same value, -1.

long f() { return -1; }
long g() { return 0xffffffff; }

And you’d be right… if you’re writing your code in Java. (Except a special number)

But, in most other popular languages ( read C++, Python, Javascript, Rust, Go, Haskell, Swift & Kotlin ), you’d get the following output:

f() : -1
g() : 4294967295

This was surprising for me and initially I thought this was a bug. And then I read the documentation for negative integer literals. In my defense though, different C++ compilers behave differently under different circumstances. For example:

  1. for x86-64 architecture, gcc and clang gives the above output but, MSVC gives -1 as the output of g().
  2. for risc-v architecture, gcc and clang gives -1 as the output of g().
  3. for [some] architecture, [gcc? clang?] gives -1 as the output of g() when the -O3 optimisation is used.

Now, coming back to the output of the very large number. In the aforementioned languages (other than Java that is), there are NO NEGATIVE INTEGER LITERALS.

Then what’s the -1 that you have in your return statement? Well, that’s a unary minus expression. It means that -1 is actually -(1) i.e. two tokens <minus> <integer literal>.

This new information helps us understand that even if a negative number is represented as a two’s-complement internally, the languages are treating 0xffffffff literal as a POSITIVE INTEGER LITERAL and since it can’t fit in a 32-bit integer, it’s promoted to a 64-bit integer and that value, 4294967295, is returned as a result.

This behavior is a language design choice (notice, how Java is handling it differently than other languages). As for why this happens, I don’t know!. All I could find was “it’s a convention to simplify the language grammar” and similar explanations. It’s probably a question for a language designer and if I find anything, I’ll probably update this post later.

I’ve added the test codes in this github repo. To add any other language, please create a pull request.
